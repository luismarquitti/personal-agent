## Summary

Permitir que o agente use modelos como Llama 3 via Ollama localmente, sem depender de APIs pagas, para desenvolvimento e testes rápidos. Inclui uma factory de LLM, scripts de setup e tutorial.

## Context

O backend usa LangGraph com `create_react_agent`. Atualmente, `app/core/graph.py` instancia `ChatGoogleGenerativeAI` de forma hardcoded. A solução abstrai isso em uma factory function compatível com `BaseChatModel`.

## User Story

Como **desenvolvedor do Personal AI Core**, quero **configurar o agente para usar Ollama localmente** para **reduzir custos de API durante desenvolvimento e iterar mais rápido nos testes.**

## Acceptance Criteria

- [ ] `app/core/llm_factory.py` implementado com suporte a `google` e `ollama`
- [ ] `app/core/graph.py` refatorado para usar a factory (sem `ChatGoogleGenerativeAI` hardcoded)
- [ ] Scripts de setup funcionam em Windows (PS1) e Linux/macOS (sh)
- [ ] Tutorial `doc/guias/local-llm-setup.md` completo
- [ ] Testes unitários de `test_llm_factory.py` passando

## Dependencies

- `langchain-ollama` (nova dependência Python)
- Ollama instalado localmente (requisito de runtime para o modo `ollama`)

## Out of Scope

- Fine-tuning de modelos
- Integração com Bedrock/Azure OpenAI
- UI de seleção de modelos

## Technical Notes

- Usar `langchain_ollama.ChatOllama` que implementa `BaseChatModel` nativamente
- Configuração via `.env`: `LLM_PROVIDER`, `OLLAMA_MODEL`, `OLLAMA_BASE_URL`
- Fallback gracioso: se Ollama offline → log warning, não crash o backend
